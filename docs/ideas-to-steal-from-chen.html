Ideas worth stealing from Kun Chen's tooling

Where his shipped tools go further than our Loop platform today. Ranked by value. Read later; pull any in when you want to improve the platform. Repos cloned at ~/chen-reference.

Value ●●● high · ●●● med · ●●● low
Effort S M L
1 Reusable worktree pool treehouse ●●●M
What
A pool of reusable, isolated worktrees — each agent gets a clean one instantly, with node_modules and build cache intact. In-use worktrees are auto-detected; returned to the pool on exit.
Our gap
We create a fresh worktree per task and reinstall deps every time. On a real repo that's minutes of cold start per task.
Payoff
Near-instant task start; far cheaper parallelism. Biggest single speedup available to us.
2 Fuller validation pipeline with auto-fix no-mistakes ●●●M–L
What
A staged gate — review → test → docs → lint → push → PR → watch CI — in a disposable worktree that auto-applies safe fixes and escalates only the rest to you.
Our gap
Our "checking" phase is evidence + one fresh-context review. No dedicated lint/docs/test-runner stages, no auto-fix loop, no CI-watch after PR.
Payoff
Catches more classes of slop before the human gate; fewer review round-trips.
3 Smarter loop: repair, retry, backoff gnhf ●●●S
What
Commit-on-success, rollback-on-failure — but commit failures are preserved for repair (not discarded), retryable hard errors back off exponentially, and agent-reported failures retry immediately.
Our gap
We roll back every failed maker run uniformly and don't retry — a partial-but-useful result or a transient API blip is thrown away.
Payoff
Fewer lost results; resilience to flaky infra. Small change to our existing rollback path.
4 Event-driven, zero-token supervision firstmate ●●●M
What
A watcher sleeps on the fleet and wakes the supervising agent only when a decision is needed — no polling. A turn-end hook blocks a "blind stop" while work is still in flight.
Our gap
The UI polls on an interval and the orchestrator drives tasks sequentially; supervision cost scales with task count.
Payoff
Cheap, quiet supervision that scales — matters once many tasks run at once.
5 Parallel crew + task shapes + ship modes firstmate ●●●L
What
One liaison spawns/supervises many agents in parallel worktrees. Two task shapes: ship (deliver a change) vs scout (investigate/plan/audit → report). Explicit project ship modes: no-mistakes / direct-PR / local-only, plus a +yolo autonomy flag.
Our gap
Our tasks are independent but concurrent execution is unproven; we have no scout/investigation task type, and ship mode is implicit (PR only if a remote exists).
Payoff
The real "30+ PRs a day" throughput; investigation tasks that leave a report with no diff.
6 Agent-ergonomic tool interfaces (AXI) axi · gh-axi ●●●L
What
Agent-native CLIs designed around token budget — benchmarked higher success at lower cost/turns than MCP (e.g. gh-axi: 100% success, $0.05, 3 turns vs GitHub MCP 87%, $0.15, 6 turns).
Our gap
Our connectors are plain MCP servers.
Payoff
Cheaper, faster, more reliable tool use — swap GitHub MCP for gh-axi, browser MCP for chrome-devtools-axi. Exploratory.
Smaller nice-to-haves
Per-run exit summary — elapsed, tokens, branch diff stats, and ready-to-run review commands at the end of every run. gnhf
Richer plan editor — Lavish turns the plan artifact into an interactive HTML doc with annotations, vs our plain editable textarea. lavish-axi
Agent-agnostic targets — run the same loop against Codex, opencode, Copilot, or any ACP target, not just Claude Code. gnhf · acpx
